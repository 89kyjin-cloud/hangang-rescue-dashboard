# CHANGELOG

## v1.0 Phase 2.6 BlankDataGuard

- 수위/방류 API 호출 성공과 수치값 공백을 분리 표시
- 후보 필드가 존재하지만 전체 기간 값이 비어 있으면 `확인실패`로 표시
- 원자료 로그에 앞/뒤 샘플과 값 현황을 추가
- 잠실철교 및 잠실대교 조석/물때 제외 로직 반영
- 로그 패널 높이를 확대해 모바일 Safari에서 원인 확인을 쉽게 조정


## v1.0 Phase 2.5 ValidValueScan

- 수위/방류량 API 응답에서 필드 존재와 값 없음 상태를 분리 표시
- 최근 행이 빈값일 때 직전 유효 숫자값을 뒤에서부터 탐색하도록 보강
- 수위/방류량이 모두 없거나 오래된 값이면 최종 판단을 `참고판정 제한 · 신뢰도 낮음`으로 표시
- 잠실철교를 잠실수중보 상류 조건에 맞춰 조석 적용 제외로 수정
- 원자료 로그에 최근 유효값 시각, 사용 필드, 건너뛴 빈값 행 수를 표시

## v1.0 Phase 2.3 FieldFix

- 수위/방류량 필드 자동탐지 강화
- 후보 필드 검증 실패 시 원자료 첫 행 전체 키와 샘플 표시
- 수위/방류량 0 임의 표시 방지 유지
- 조석 API `GetTideFcstTimeApiService` 유지
- `index.html`의 `app.js` 캐시 버전 `fieldfix`로 변경
